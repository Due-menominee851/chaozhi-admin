# 模块需求

## 1. 基本信息

- 模块中文名：物料管理
- 模块英文名：Material
- 路径名：`material`
- 模块定位：基础资料模块

## 2. 功能范围

需要生成：

- 分页查询
- 新增
- 编辑
- 删除

当前不需要：

- 详情页
- 导入导出
- 批量操作
- 树结构
- 主子表

## 3. 字段定义

- `name`，string，必填，可搜索，备注=物料名称
- `code`，string，必填，可搜索，备注=物料编码
- `spec`，string，非必填，可搜索，备注=规格型号
- `unit`，string，非必填，不可搜索，备注=单位
- `status`，string，必填，可搜索，备注=状态，枚举=`ENABLE=启用, DISABLE=停用`
- `sort`，number，非必填，不可搜索，备注=排序
- `remark`，text，非必填，不可搜索，备注=备注

## 4. 业务规则

- `code` 必须唯一
- 列表默认按 `id` 倒序
- `status` 在搜索区和表单中使用下拉
- `status` 在列表中使用 `tag` 展示
- `remark` 只在表单展示，不要求放在列表主列
- 新增和编辑共用同一个弹窗

## 5. 数据库要求

- 表名：`t_material`
- 使用默认主键：`id`
- 使用默认时间字段：`create_time`、`update_time`

## 6. 前端要求

- 页面标题：物料管理
- 菜单图标：`lucide:package`
- 路由排序：`1100`
- 需要国际化
- 使用当前项目已有 CRUD 页面结构

列表筛选条件至少支持：

- `name`
- `code`
- `spec`
- `status`

## 7. 后端要求

- 生成标准 CRUD 分层代码
- DTO 必填字段加校验注解
- 列表查询支持按 `name`、`code`、`spec`、`status` 筛选
- 删除时如果数据不存在，应返回业务异常，不要静默成功
- 错误码优先使用现有 `CommonErrCode`

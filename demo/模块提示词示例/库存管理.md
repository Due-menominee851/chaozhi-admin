# 模块需求

## 1. 基本信息

- 模块中文名：库存管理
- 模块英文名：Inventory
- 路径名：`inventory`
- 模块定位：库存查询与库存预警展示

## 2. 功能范围

需要生成：

- 库存分页列表
- 查询筛选
- 查看库存详情
- 查看库存流水
- 手动刷新库存

当前不需要：

- 新增库存
- 手工编辑库存余额
- 导入导出
- 盘点功能
- 调拨功能
- WebSocket 实时推送

## 3. 字段定义

- `materialCode`，string，必填，可搜索，备注=物料编码
- `materialName`，string，必填，可搜索，备注=物料名称
- `spec`，string，非必填，可搜索，备注=规格型号
- `unit`，string，非必填，不可搜索，备注=单位
- `warehouseCode`，string，必填，可搜索，备注=仓库编码
- `warehouseName`，string，必填，可搜索，备注=仓库名称
- `currentStock`，number，非必填，不可搜索，备注=当前库存
- `availableStock`，number，非必填，不可搜索，备注=可用库存
- `lockedStock`，number，非必填，不可搜索，备注=锁定库存
- `safetyStock`，number，非必填，不可搜索，备注=安全库存
- `warningStatus`，string，非必填，可搜索，备注=预警状态，枚举=`NORMAL=正常, LOW=库存不足, ZERO=无库存`
- `updateTime`，datetime，非必填，不可搜索，备注=最后更新时间

## 4. 业务规则

- 列表默认按 `updateTime` 倒序
- `warningStatus` 在搜索区使用下拉，在列表中使用 `tag`
- `currentStock`、`availableStock`、`lockedStock`、`safetyStock` 右对齐显示
- 如果后端已返回 `warningStatus`，前端优先使用接口值
- 若后端未返回 `warningStatus`，前端可按以下口径兜底展示：
  - `availableStock <= 0` 显示 `ZERO`
  - `availableStock > 0 && availableStock <= safetyStock` 显示 `LOW`
  - 其他显示 `NORMAL`
- 列表项默认带 `id`，作为表格 `row-key`
- 支持查看库存详情和库存流水
- 顶部“刷新库存”点击后，调用后端刷新接口，再重新请求列表并更新页面上的最后刷新时间
- 当前版本“查看库存流水”优先跳转到独立的库存流水页面，并自动带入当前行筛选条件；只有在没有独立流水页面时才降级为本页弹窗展示
- 当前版本库存详情优先使用抽屉或弹窗展示，不做复杂多标签详情页

## 5. 前端要求

- 页面标题：库存管理
- 菜单图标：`lucide:boxes`
- 路由排序：`1400`
- 需要国际化
- 页面结构包含“查询区 + 顶部操作区 + 表格”
- 顶部操作区至少包含：刷新库存
- 列表操作列至少包含：详情、库存流水
- 操作按钮统一使用 `Button type="link"`，并按语义区分颜色
- 多个操作按钮必须使用 `Space`
- 详情优先使用弹窗或抽屉，不要一开始就做复杂多标签详情页

列表筛选条件至少支持：

- `materialCode`
- `materialName`
- `spec`
- `warehouseName`
- `warningStatus`

## 6. 数据库要求

库存查询属于结果展示模块，默认至少需要一张库存结果表：

- 主表：`t_inventory`

如果当前项目需要持久化安全库存等规则，可继续在该表扩展字段；本次不强制拆分更多主数据表。

## 7. 后端要求

- 生成库存查询模块的完整分层代码
- 分页查询接口返回 `PageVO`
- 详情接口返回单条库存详情
- 刷新接口使用：
  - `POST /inventory/refresh`
- `POST /inventory/refresh` 表示触发一次后端库存刷新动作，不是单纯前端重查
- `warningStatus` 优先由后端计算并返回
- 如果后端未返回 `warningStatus`，前端才允许按文档规则做兜底展示
- 如果项目已有独立库存流水模块，`GET /inventory/{id}/logs` 可直接复用库存流水查询结果
- 错误码优先使用现有 `CommonErrCode`

## 8. 接口对接要求

默认按以下接口对接：

- `GET /inventory/page`
- `GET /inventory/{id}`
- `GET /inventory/{id}/logs`
- `POST /inventory/refresh`

返回约定：

- `GET /inventory/page` 的列表项默认带 `id`
- `GET /inventory/{id}` 返回单条库存详情
- `GET /inventory/{id}/logs` 返回该库存对应的流水列表
- 如果当前项目已有独立 `inventory-log` 页面，优先跳转并带筛选参数，而不是重复造一套页面
- `POST /inventory/refresh` 表示触发一次后端库存刷新动作，不是单纯前端重新拉取列表
- `POST /inventory/refresh` 成功后提示“刷新成功”，并立即刷新列表

# 模块需求

## 1. 基本信息

- 模块中文名：库存流水
- 模块英文名：InventoryLog
- 路径名：`inventory-log`
- 模块定位：库存变动查询与追溯

## 2. 功能范围

需要生成：

- 库存流水分页列表
- 列表筛选
- 查看流水详情

当前不需要：

- 新增
- 编辑
- 删除
- 导入导出
- 图表分析

## 3. 字段定义

- `bizNo`，string，必填，可搜索，备注=业务单号
- `bizType`，string，必填，可搜索，备注=业务类型，枚举=`PURCHASE_IN=采购入库, SALE_OUT=销售出库, ADJUST=库存调整, TRANSFER=库存调拨`
- `materialCode`，string，必填，可搜索，备注=物料编码
- `materialName`，string，必填，可搜索，备注=物料名称
- `spec`，string，非必填，可搜索，备注=规格型号
- `warehouseName`，string，必填，可搜索，备注=仓库名称
- `changeType`，string，必填，可搜索，备注=变动方向，枚举=`IN=入库, OUT=出库`
- `changeQty`，number，非必填，不可搜索，备注=变动数量
- `beforeStock`，number，非必填，不可搜索，备注=变动前库存
- `afterStock`，number，非必填，不可搜索，备注=变动后库存
- `operatorName`，string，非必填，可搜索，备注=操作人
- `operateTime`，datetime，非必填，可搜索，备注=操作时间
- `remark`，string，非必填，不可搜索，备注=备注

## 4. 业务规则

- 列表默认按 `operateTime` 倒序
- `bizType`、`changeType` 在搜索区使用下拉，在列表中使用 `tag`
- 数量字段右对齐显示
- `changeType=IN` 时数量偏正向色展示
- `changeType=OUT` 时数量偏警示色展示
- 列表项默认带 `id`，作为表格 `row-key`
- 详情中应展示业务来源、物料信息、仓库信息、变动前后库存、操作时间、操作人
- 如果从库存管理页面进入，可允许自动带入 `materialCode`、`warehouseName` 等默认筛选条件

## 5. 前端要求

- 页面标题：库存流水
- 菜单图标：`lucide:history`
- 路由排序：`1410`
- 需要国际化
- 页面结构包含“查询区 + 表格 + 操作区”
- 操作区至少包含：查看详情
- 多个操作按钮必须使用 `Space`
- 如果存在路由参数默认筛选条件，首次加载应自动查询
- 详情优先使用弹窗或抽屉，避免过重的多页签详情页

列表筛选条件至少支持：

- `bizNo`
- `bizType`
- `materialCode`
- `materialName`
- `warehouseName`
- `changeType`
- `operatorName`
- `operateTime` 起止范围

## 6. 数据库要求

库存流水模块默认至少需要一张流水表：

- 主表：`t_inventory_log`

默认字段：

- 包含 `id`、`create_time`、`update_time`

## 7. 后端要求

- 生成库存流水查询模块的完整分层代码
- 分页查询接口返回 `PageVO`
- 详情接口返回单条流水详情对象
- 列表默认按 `operateTime` 倒序
- `bizType`、`changeType` 枚举值与系统总览中已有业务类型保持一致
- 详情至少包含业务来源、物料信息、仓库信息、变动前后库存、操作信息
- 错误码优先使用现有 `CommonErrCode`

## 8. 接口对接要求

默认按以下接口对接：

- `GET /inventory-log/page`
- `GET /inventory-log/{id}`

返回约定：

- `GET /inventory-log/page` 的列表项默认带 `id`
- `GET /inventory-log/{id}` 返回单条流水详情对象
- 详情至少包含业务来源、物料信息、仓库信息、变动前后库存、操作信息

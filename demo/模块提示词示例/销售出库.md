# 模块需求

## 1. 基本信息

- 模块中文名：销售出库
- 模块英文名：StockOutOrder
- 路径名：`stock-out-order`
- 模块定位：销售或业务领料后的仓库出库单据

## 2. 功能范围

需要生成：

- 出库单分页列表
- 列表筛选
- 新增
- 编辑
- 删除
- 查看详情
- 提交
- 出库确认
- 作废

当前不需要：

- 导入导出
- 打印
- 工作流审批
- 附件上传
- 复杂物流单号管理

## 3. 业务背景

出库单用于处理销售发货或其他业务出库。
用户选择来源单或物料后，系统自动带出基础信息和库存相关字段，仓库人员填写本次出库数量并完成确认。

## 4. 状态流转

- `DRAFT`：草稿
- `PENDING`：待出库
- `COMPLETED`：已出库
- `CANCELLED`：已作废

流转规则：

- `DRAFT -> PENDING`：提交
- `PENDING -> COMPLETED`：出库确认
- `DRAFT -> CANCELLED`：作废
- `PENDING -> CANCELLED`：作废

状态限制：

- 只有 `DRAFT` 允许编辑、删除、提交
- 只有 `PENDING` 允许出库确认
- `DRAFT`、`PENDING` 允许作废
- `COMPLETED`、`CANCELLED` 不允许再编辑

## 5. 主表字段定义

- `outNo`，string，非前端录入，可搜索，备注=出库单号，由后端自动生成
- `sourceOrderNo`，string，必填，可搜索，备注=来源单号
- `warehouseName`，string，必填，可搜索，备注=出库仓库
- `customerName`，string，非必填，可搜索，备注=客户名称
- `operatorName`，string，非必填，可搜索，备注=经办人
- `outDate`，date，必填，可搜索，备注=出库日期
- `status`，string，必填，可搜索，备注=状态，枚举=`DRAFT=草稿, PENDING=待出库, COMPLETED=已出库, CANCELLED=已作废`
- `remark`，text，非必填，不可搜索，备注=备注

## 6. 明细字段定义

- `materialId`，number，必填，不可搜索，备注=物料ID
- `materialCode`，string，必填，不可搜索，备注=物料编码
- `materialName`，string，必填，不可搜索，备注=物料名称
- `spec`，string，非必填，不可搜索，备注=规格型号
- `unit`，string，非必填，不可搜索，备注=单位
- `stockQty`，number，非必填，不可搜索，备注=当前库存
- `availableQty`，number，非必填，不可搜索，备注=可用库存
- `outQty`，number，必填，不可搜索，备注=本次出库数量
- `remark`，string，非必填，不可搜索，备注=明细备注

## 7. 业务规则

- 列表默认按 `id` 倒序
- `status` 在搜索区使用下拉，在列表中使用 `tag`
- 明细至少保留 1 行
- `outQty` 必须大于 0
- `outNo` 由后端自动生成，新增时前端不展示输入框，详情和编辑时只读展示
- 当前版本 `sourceOrderNo` 使用自由文本输入，仅作为业务来源号记录，不强制关联固定来源模块
- 当前版本 `warehouseName`、`customerName`、`operatorName` 使用自由文本输入，不接主数据下拉
- 明细必须通过选择物料带出，不允许手工自由输入 `materialCode`、`materialName`、`spec`、`unit`
- 来源单或库存带出的 `materialCode`、`materialName`、`spec`、`unit`、`stockQty`、`availableQty` 默认只读
- `availableQty` 以后端返回为准，前端不自行计算
- `stockQty`、`availableQty` 必须由后端库存接口返回，不允许前端自行推导
- 如果后端返回 `availableQty`，前端应限制 `outQty <= availableQty`
- 前端必须校验 `outQty <= availableQty`
- 如果可用库存不足，应给出明显表单提示
- 当前版本允许手动新增明细行，但每一行都必须先选择物料，再由后端返回库存相关字段
- 详情接口返回“主表信息 + items 明细数组”
- `items` 每行必须带 `id`，前端 `row-key` 使用该 `id`

## 8. 前端要求

- 页面标题：销售出库
- 菜单图标：`lucide:truck`
- 路由排序：`1500`
- 需要国际化
- 页面结构包含“查询区 + 表格 + 操作区”
- 列表操作至少包含：编辑、删除、详情、提交、出库确认、作废
- 操作按钮统一使用 `Button type="link"`，并按语义区分颜色
- 多个操作按钮必须使用 `Space`
- 新增/编辑使用自定义主子表弹窗

列表筛选条件至少支持：

- `outNo`
- `sourceOrderNo`
- `warehouseName`
- `customerName`
- `status`
- `outDate` 起止范围

## 9. 数据库要求

需要两张表：

- 主表：`t_stock_out_order`
- 明细表：`t_stock_out_order_item`

默认字段：

- 主表包含 `id`、`create_time`、`update_time`
- 明细表包含 `id`、`order_id`、`create_time`、`update_time`

## 10. 后端要求

- 生成主表和明细的完整分层代码
- 主表分页查询接口返回 `PageVO`
- 查询详情时返回主表 + 明细
- 新增、编辑采用事务处理
- `outNo` 由后端生成
- 所有状态流转都必须做状态校验
- 出库确认成功后应驱动库存扣减，并写入库存流水
- 如果当前版本使用物料库存接口选明细，则后端需提供可用库存查询能力
- 删除不存在的数据时返回业务异常，不要静默成功
- 错误码优先使用现有 `CommonErrCode`

## 11. 接口对接要求

默认按以下接口对接：

- `GET /stock-out-order/page`
- `GET /stock-out-order/{id}`
- `POST /stock-out-order`
- `PUT /stock-out-order`
- `DELETE /stock-out-order/{id}`
- `PUT /stock-out-order/{id}/submit`
- `PUT /stock-out-order/{id}/confirm`
- `PUT /stock-out-order/{id}/cancel`

如果存在来源单查询，默认要求返回：

- 主表信息
- `items` 明细数组
- 明细数组每行带 `id`

如果当前版本没有来源单查询接口，则至少需要一个物料库存查询接口用于选择明细：

- `GET /inventory/material-options?keyword=xxx&warehouseName=xxx`

返回约定：

- 返回物料基础信息与当前库存信息
- 每行至少包含 `materialId`、`materialCode`、`materialName`、`spec`、`unit`、`stockQty`、`availableQty`
